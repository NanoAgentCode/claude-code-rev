# Claude Code Rev 打包发布指南

## 1. 目的

本文档用于指导你将当前项目打成可安装的 npm 包（`.tgz`），并在本地验证安装与运行。

适用场景：

- 本地试装验证
- 内部分享测试包
- 为后续 npm 正式发布做预演

---

## 2. 环境要求

- Bun >= `1.3.5`
- Node.js >= `24`（低版本可能仅告警，但建议满足要求）
- npm 可用（用于 `npm pack` 和安装测试）

建议先检查版本：

```bash
bun --version
node --version
npm --version
```

---

## 3. 打包前检查

在项目根目录执行：

```bash
bun install
bun run verify:baseline
```

说明：

- `verify:baseline` 会执行版本检查与类型检查
- 若这里失败，请先修复再打包

---

## 4. 生成安装包（.tgz）

在项目根目录执行：

```bash
npm pack
```

成功后会在根目录生成类似文件：

```text
anthropic-ai-claude-code-999.0.0-restored.tgz
```

---

## 5. 本地安装验证（推荐）

建议在一个全新空目录验证，不要直接在项目目录验证。

### 5.1 创建测试目录并初始化

```bash
mkdir pack-test
cd pack-test
npm init -y
```

### 5.2 安装刚打好的包

将下面路径替换为你的实际路径：

```bash
npm i "C:\Users\13439\Desktop\workspace\claude-code-rev\anthropic-ai-claude-code-999.0.0-restored.tgz"
```

### 5.3 验证 CLI 可执行

```bash
npx claude-code-rev --version
```

预期输出类似：

```text
999.0.0-restored (Claude Code)
```

---

## 6. 常见问题

## 6.1 `EBADENGINE` 警告

现象：

- 安装时提示 Node 版本不满足（例如本机是 Node 22，包要求 Node 24+）

处理：

- 建议升级到 Node 24+
- 仅本地验证时，如果功能可运行，可暂时忽略告警

## 6.2 找不到 `bun`

现象：

- 执行 `claude-code-rev` 时提示无法启动 bun

处理：

- 安装 Bun 并确认 `bun --version` 可用
- 确认系统 PATH 中包含 Bun

## 6.3 打包成功但运行失败

建议检查：

- `bun run version` 在源码目录是否可运行
- `package.json` 中 `bin` 与入口文件是否匹配
- 依赖是否安装完整（重新执行 `bun install`）

---

## 7. 可选：发布到 npm（正式）

如果你后续要正式发布公网 npm，请按以下步骤：

1. 确认包名可用、版本号规范；
2. 使用 Node 24+ 做完整验证；
3. 登录 npm：

```bash
npm login
```

4. 发布：

```bash
npm publish --access public
```

发布前建议先做一次 dry-run：

```bash
npm publish --dry-run
```

---

## 8. 建议发布检查清单

- [ ] `bun install` 成功
- [ ] `bun run verify:baseline` 成功
- [ ] `npm pack` 成功并产出 `.tgz`
- [ ] 在新目录中 `npm i <tgz>` 成功
- [ ] `npx claude-code-rev --version` 输出正常
- [ ]（正式发布）Node 版本满足要求且无关键告警
