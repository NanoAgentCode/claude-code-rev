# Claude Code Rev 项目运行调度流程报告

## 1. 报告目标

从应用流程角度描述项目从启动到退出的端到端调度路径，帮助你理解运行时行为与问题定位路径。

---

## 2. 全局调度图

```mermaid
flowchart TD
    S0[进程启动] --> S1[Bootstrap初始化]
    S1 --> S2{参数分流}
    S2 -->|快速路径| F1[Version/Daemon/Bridge等]
    S2 -->|完整路径| N1[加载main主流程]
    N1 --> N2[配置/认证/策略初始化]
    N2 --> N3[命令/工具/MCP/插件装配]
    N3 --> N4[进入REPL或非交互执行]
    N4 --> N5[QueryEngine单轮循环]
    N5 --> N6[状态持久化与遥测]
    N6 --> N4
```

---

## 3. 启动调度

- 阶段 A：最小初始化（宏、异常处理）
- 阶段 B：CLI 参数识别与快速路径判定
- 阶段 C：未命中快速路径则加载主流程并完成能力装配

---

## 4. 主流程调度

```mermaid
flowchart LR
    I1[环境配置] --> I2[认证准备]
    I2 --> I3[策略限制加载]
    I3 --> I4[插件技能初始化]
    I4 --> I5[MCP连接]
    I5 --> I6[工具池构建]
    I6 --> I7[状态初始化]
```

---

## 5. 会话闭环调度

```mermaid
sequenceDiagram
    participant U as 用户
    participant R as 交互层
    participant Q as QueryEngine
    participant M as 模型
    participant T as 工具
    participant X as 外部服务

    U->>R: 输入
    R->>Q: 组装请求
    Q->>M: 推理请求
    M-->>Q: 消息/工具意图
    Q->>T: 执行工具
    T->>X: 外部调用(可选)
    X-->>T: 结果返回
    T-->>Q: 归一化结果
    Q-->>R: 输出与状态更新
```

---

## 6. 异常与降级

- 认证异常：刷新或要求重新登录
- 权限拒绝：返回可解释原因
- 外部依赖失败：重试、降级或熔断
- 不可恢复错误：安全退出并记录

---

## 7. 调度优化建议

- 建立启动分流清单与最小依赖图
- 强化端到端调度指标（启动耗时、单轮时延、工具成功率）
- 建立故障定位 SOP（入口层 -> 编排层 -> 工具层 -> 外部层）
